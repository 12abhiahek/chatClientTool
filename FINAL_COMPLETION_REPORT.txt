                              TROUBLESHOOTING GUIDE
═══════════════════════════════════════════════════════════════════════════════════

ISSUE: Build fails
  Solution:
    1. Ensure Maven is installed: mvn --version
    2. Run: mvn clean
    3. Try again: mvn clean package -DskipTests

ISSUE: Tests fail
  Solution:
    1. Rebuild first: mvn clean
    2. Then test: mvn test -Dtest=WeatherToolCacheTests
    3. Check for test output in target/surefire-reports/

ISSUE: Application won't start
  Solution:
    1. Check if port 8080 is available
    2. Look at console logs for errors
    3. Verify JDK 17 or higher is installed

ISSUE: Cache not working (still seeing multiple API calls)
  Solution:
    1. Check CacheConfig.java exists in config folder
    2. Restart application
    3. Verify logs show cache indicators
    4. Check that @Cacheable annotation is in Weathertool.java

ISSUE: Cache not expiring after TTL
  Solution:
    1. Restart application (in-memory cache clears)
    2. Or modify CacheConfig.java to test with shorter TTL
    3. Remember: cache expires after set duration (default 30 min)

ISSUE: Error responses getting cached
  Solution:
    1. This should NOT happen (errors excluded by "unless" condition)
    2. Check the @Cacheable annotation includes "unless" clause
    3. Verify Weathertool.java has correct annotation
